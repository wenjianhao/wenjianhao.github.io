# Wenjian Hao Website

This repository now serves a plain static website for:

- `https://www.wenjianhao.com`

It is deployed directly from the repository root with GitHub Pages.

## 1. Local development

Because this is a static site, you do not need Hugo.
To regenerate the HTML from the source content:

```bash
PYTHONDONTWRITEBYTECODE=1 python3 scripts/build_static_site.py
```

Then preview with any static file server, for example:

```bash
python3 -m http.server
```

Open: `http://localhost:8000`

## 2. Main files to edit

- `scripts/build_static_site.py`: static site generator
- `css/site.css`: site styling
- `content/`: source material for papers, projects, blog, and miscellaneous pages
- `static/`: static assets such as images and favicon files

## 3. GitHub Pages deployment

Use GitHub Pages with:

1. `Settings -> Pages`
2. Source: `Deploy from a branch`
3. Branch: `main`
4. Folder: `/ (root)`

This repo includes `.nojekyll` so GitHub Pages serves the static files directly.

## 4. Custom domain

`CNAME` is set to:

```txt
www.wenjianhao.com
```

At your DNS provider, set:

- `www` as `CNAME` to `<your-github-username>.github.io`

For the apex (`wenjianhao.com`), either:

- redirect to `www.wenjianhao.com`, or
- use GitHub Pages apex A records (if you prefer no redirect).

## 5. Notes

- This is no longer a Hugo site.
- It is a static GitHub Pages site generated by `scripts/build_static_site.py`.

## 6. Visitor analytics setup (GoatCounter)

The homepage visitor map is ready to use real GoatCounter data, but it will stay in preview mode until you add the required GitHub secrets.

### Step 1. Create or open your GoatCounter site

Go to [GoatCounter](https://www.goatcounter.com/) and create a site if you do not already have one.

You will need:

- your GoatCounter site code
- an API access token

### Step 2. Add GitHub repository secrets

In GitHub:

1. open your repository
2. go to `Settings`
3. go to `Secrets and variables -> Actions`
4. click `New repository secret`

Add these secrets:

#### `GOATCOUNTER_CODE`

Use either:

- just the site code, for example:

```txt
wenjianhao
```

or a full GoatCounter endpoint, for example:

```txt
https://wenjianhao.goatcounter.com/count
```

#### `GOATCOUNTER_API_KEY`

Paste your GoatCounter API token here.

#### Optional: `GOATCOUNTER_ENDPOINT`

Only add this if you want to override the default API base URL.

Example:

```txt
https://wenjianhao.goatcounter.com
```

### Step 3. Run the workflow

After adding the secrets:

1. go to `Actions`
2. open `Refresh Visitor Data`
3. click `Run workflow`

This workflow will:

- fetch real visitor totals and country-level data from GoatCounter
- write them to `static/data/visitors.json`
- rebuild the site
- push the updated static files back to the repository

### Step 4. Verify the homepage

Once the workflow finishes, the visitor section should stop showing:

```txt
Preview mode. Real analytics will appear after GoatCounter is connected.
```

and should instead show:

- real views
- real visitors
- real country count
- real country-level map dots and labels

# Wenjian Hao Website

This repository now serves a plain static website for:

- `https://www.wenjianhao.com`

It is deployed directly from the repository root with GitHub Pages.

## 1. Local development

Because this is a static site, you do not need Hugo.
To regenerate the HTML from the source content:

```bash
PYTHONDONTWRITEBYTECODE=1 python3 scripts/build_static_site.py
```

Then preview with any static file server, for example:

```bash
python3 -m http.server
```

Open: `http://localhost:8000`

## 2. Main files to edit

- `scripts/build_static_site.py`: static site generator
- `css/site.css`: site styling
- `content/`: source material for papers, projects, blog, and miscellaneous pages
- `static/`: static assets such as images and favicon files

## 3. GitHub Pages deployment

Use GitHub Pages with:

1. `Settings -> Pages`
2. Source: `Deploy from a branch`
3. Branch: `main`
4. Folder: `/ (root)`

This repo includes `.nojekyll` so GitHub Pages serves the static files directly.

## 4. Custom domain

`CNAME` is set to:

```txt
www.wenjianhao.com
```

At your DNS provider, set:

- `www` as `CNAME` to `<your-github-username>.github.io`

For the apex (`wenjianhao.com`), either:

- redirect to `www.wenjianhao.com`, or
- use GitHub Pages apex A records (if you prefer no redirect).

## 5. Notes

- This is no longer a Hugo site.
- It is a static GitHub Pages site generated by `scripts/build_static_site.py`.
